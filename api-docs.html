Save Load API For Unreal Engine Docs Help

API Overview

Introduction

The Save Load API for Unreal Engine is designed to give developers more control while simplifying the process of saving and loading data in Unreal Engine projects. The built-in save system in Unreal Engine has several limitations: it is often project-specific, less extensible, more complicated, and can be prone to breaking during significant updates. Additionally, it tends to be slower to load and consumes more disk space.

This API was created to address these challenges and is currently in use for my commercial game project, Coffee Shop Simulator. You can trust the quality of this API since it was specifically developed to be used in a real-world game project.

API Versioning

Current Version

The current version of the API is 1.0.0 This indicates that the API is in its first stable release.

Versioning Information

In the context of APIs, versioning is an important aspect that allows the API developers to make improvements, introduce new features, and fix bugs, while also ensuring backward compatibility with older versions. Incrementing the version number allows developers to track and manage different stages of the API life cycle.

Here's a brief explanation of the version number system:

  • Major Version (1 in 1.0.0): This number is incremented when there are incompatible changes in the API. Using version 1 suggests that the API has a stable release now.

  • Minor Version (first 0 in 1.0.0): This number is incremented for backward-compatible new features and improvements. The number '0' indicates no minor updates have been released since the first stable version.

  • Patch or Revision number (last 0 in 1.0.0): This number, is incremented for backward-compatible bug fixes.
