Save Load API For Unreal Engine Docs Help

USaveLoadManager

This class is responsible for managing saving and loading data.

inline static FString DefaultSaveFileName;

  • Brief: This is the default save file name for your game. You can change this value, but make sure to delete your last save before doing so. This FString value is also private, therefore you will have to use the GetDefaultSaveFileName() method in order to get it.

static FString PrepareFilePath(const FString& FileName, ESaveFileFormat SaveFileFormat);

  • Brief: Prepare the file path for saving or loading data.

  • FileName: The name of the file.

  • SaveFileFormat: The format of the save file.

  • Returns: The full file path for the specified file name and format.

static bool DoesFileExist(const FString& FilePath);

  • Brief: Checks if a file exists at the given file path.

  • FilePath: The file path to check.

  • Returns: True if the file exists, false otherwise.

static void DeleteFile(const FString& FileName);

  • Brief: Deletes a file with the specified file name.

  • FileName: The name of the file to be deleted.

static FString GetDefaultSaveFileName();

  • Brief: This method returns the default file name for saving data. The default file name is stored in the static member variable DefaultSaveFileName.

  • Returns: An FString representing the default file name for saving data.

Note: Return value can be changed by modifying the DefaultSaveFileName variable.

static bool SaveData(const FString& Key, const TArray<uint8>& Data, EDataType DataType, const FString& SaveFilePath);

  • Brief: This method saves the given data to a file at the specified path. If the file already exists, the data will be appended to the existing data. If the file does not exist, a new file will be created.

  • Key: The key associated with the data. This is used to identify and retrieve the data when loading.

  • Data: The data to be saved. This should be an array of uint8 values.

  • DataType The type of data being saved.

  • SaveFilePath: The path to the file where the data will be saved. This should be a valid file path including the file name and extension.

  • Returns: true if the data was successfully saved, false otherwise.

static bool LoadData(const FString& Key, TArray<uint8>& OutData, EDataType& OutDataType, const FString& SaveFilePath);

  • Brief: This method loads the given data from a file at the specified path.

  • Key: The key used to identify the data in the save file.

  • OutData: The output array that will contain the loaded data.

  • OutDataType: The output data type of the loaded data.

  • SaveFilePath: The file path of the save file.

  • Returns: True if the data was successfully loaded, false otherwise.

static bool DeleteData(const FString& Key, const FString& SaveFilePath);

  • Brief: This method deletes a data entry with the specified key from the save file located at the provided file path. The method loads the save file into memory, removes the data entry with the specified key, and then serializes the remaining entries back to the file.

  • Key: The key of the data entry to be deleted.

  • SaveFilePath: The file path to the save file.

  • Returns: True if the data entry is successfully deleted, false otherwise.

  • Value: The float value to convert.

  • Returns: The byte array representation of the float value.

  • Brief: Converts a byte array to a float value.

  • ByteArray: The byte array to be converted to a float value.

  • Returns: The resulting float value from the byte array.

  • Brief: Converts a byte array to a boolean value.

  • ByteArray: The array of bytes to convert to a boolean value.

  • Returns: The boolean value converted from the byte array.

static TArray<uint8> EnumToByteArray(uint8 EnumValue)

  • Brief: This method takes an enumeration value as input and converts it to a byte array representation.

  • EnumValue: The enumeration value to convert to a byte array.

  • Returns: A byte array representation of the given enumeration value.

Warning: This method can take uint8, uint16, uint32 and uint64 as a parameter, however since the Unreal Engine Blueprints only support uint8 variable type, other variable types(uint16, uint32 and uint64) cannot be used on Unreal Engine Blueprints. If you do not intend on using Unreal Engine Blueprints for saving Enum Data, then you should be fine with using any of these variable types.
